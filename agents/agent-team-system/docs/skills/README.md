## 使用指南

### 新模块开发

1. **阅读 SOP** → `phase3_development_sop.md`
2. **遵循流程** → 6 个阶段逐步执行
3. **使用检查清单** → 确保质量
4. **参考技能文档** → `module_development_skill.md`

### 功能迭代

1. **评估变更范围**
2. **更新设计文档**
3. **修改代码**
4. **运行测试**
5. **更新文档**
6. **提交代码**

### 代码重构

1. **确保测试覆盖**
2. **小步重构**
3. **频繁运行测试**
4. **保持功能不变**

## 质量标准

### 代码质量

- ✅ PEP 8 规范
- ✅ 类型注解完整
- ✅ 文档字符串完整
- ✅ 无 LSP 错误

### 测试质量

- ✅ 单元测试覆盖率 >= 80%
- ✅ 所有测试通过 (100%)
- ✅ 深度测试覆盖边界情况
- ✅ 执行时间 < 5 秒

### 文档质量

- ✅ 设计文档 >= 1000 行
- ✅ 至少 3 个 Mermaid 流程图
- ✅ 使用示例可运行
- ✅ API 文档完整

## 最佳实践

### 设计先行

```
❌ 错误做法：直接写代码
✅ 正确做法：先设计，后实现
```

### 测试驱动

```
❌ 错误做法：写完代码再写测试
✅ 正确做法：先写测试，后实现功能
```

### 小步提交

```
❌ 错误做法：一次性提交大量代码
✅ 正确做法：小步快跑，频繁提交
```

### 文档同步

```
❌ 错误做法：代码写完再补文档
✅ 正确做法：代码与文档同步更新
```

## 工具推荐

### 开发工具

- **Python**: 3.12+
- **pytest**: 测试框架
- **pytest-asyncio**: 异步测试
- **pytest-cov**: 覆盖率

### 代码质量

- **black**: 代码格式化
- **mypy**: 类型检查
- **pylint**: 代码审查
- **flake8**: 代码风格

### 文档工具

- **Mermaid**: 流程图
- **Markdown**: 文档格式
- **pdoc**: API 文档
