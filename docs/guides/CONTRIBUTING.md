# 🤝 贡献指南

感谢您对 Agent School 项目的关注！我们欢迎各种形式的贡献。

## 📋 贡献方式

### 🐛 报告问题
- 使用 GitHub Issues 模板
- 提供详细的错误描述和重现步骤
- 包含环境信息（操作系统、Node.js/Python 版本等）

### 💡 提出功能建议
- 清晰描述您想要的功能
- 说明该功能解决的问题
- 如果可能，提供实现思路

### 🔧 代码贡献
1. Fork 项目
2. 创建功能分支
3. 编写代码和测试
4. 提交 Pull Request

## 📝 开发规范

### 代码风格
- **JavaScript/TypeScript**: 遵循 ESLint 配置
- **Python**: 遵循 PEP 8 规范
- 使用有意义的变量名和函数名

### 提交信息格式
```
<类型>(<范围>): <描述>

[可选的正文]

[可选的脚注]
```

**类型包括：**
- `feat`: 新功能
- `fix`: 修复 bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 重构代码
- `test`: 测试相关
- `chore`: 构建过程或辅助工具变动

### 示例提交信息
```
feat(agents): 添加对话记忆功能

实现了基于 Redis 的对话历史存储，
支持长期记忆和短期记忆分离。

Closes #123
```

## 🧪 测试要求

### 提交前检查清单
- [ ] 代码通过所有测试
- [ ] 新增功能有相应的测试用例
- [ ] 文档已更新
- [ ] 代码符合风格规范

### 运行测试
```bash
# 运行全部测试
npm test

# 运行特定测试
npm test -- --grep="agent"

# 生成测试覆盖率报告
npm run test:coverage
```

## 📚 文档贡献

### 文档结构
- `docs/` - 技术文档
- `BEGINNER_GUIDE.md` - 新手指南
- `README.md` - 项目主文档

### 文档编写规范
- 使用 Markdown 格式
- 保持语言简洁明了
- 包含实际代码示例
- 提供清晰的步骤说明

## 🎯 Pull Request 流程

1. **创建 PR**
   - 填写 PR 模板
   - 关联相关 Issue
   - 描述变更内容

2. **代码审查**
   - 等待维护者审查
   - 根据反馈进行修改
   - 保持沟通畅通

3. **合并**
   - 通过所有检查后合并
   - 维护者会处理后续发布
